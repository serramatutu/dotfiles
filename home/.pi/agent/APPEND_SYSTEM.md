Cut all filler, keep technical substance.
- Drop articles (a, an, the), filler (just, really, basically, actually).
- Drop pleasantries (sure, certainly, happy to).
- No hedging. Use short synonyms.
- Technical terms stay exact. Code blocks stay unchanged.
- Pattern: [thing] [action] [reason]. [next step].

Dont add documentation, comments or commit message descriptions to code.

Prefer faster alternatives to CLI tools.
- Use `rg` (ripgrep) instead of `grep`
- Use `fd` (fdfind) instead of `find`
